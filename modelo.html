📊 Anatomía del Modelo Relacional

El modelo relacional organiza los datos en estructuras bidimensionales. Es la base de MySQL. A continuación explicamos sus componentes clave:

📁 Tabla

Es la estructura principal. Representa una "entidad" o concepto del mundo real (ej. Tabla `Usuarios`, Tabla `Productos`).

📋 Registro (Fila / Tupla)

Es cada fila individual dentro de la tabla. Contiene los datos específicos de un solo objeto o individuo (ej. Los datos del usuario Juan).

🔍 Campo (Columna / Atributo)

Es cada columna de la tabla. Define el tipo de dato que se va a guardar (ej. `nombre`, `correo`, `fecha_registro`).

🔑 El Poder de las Claves (Keys)

Para conectar las tablas entre sí y garantizar que no haya datos repetidos de forma ambigua, usamos llaves:

🔑 Clave Primaria (PK - Primary Key): Es un campo único e irrepetible en toda la tabla que identifica de forma inequívoca a cada registro. Por ejemplo, el número de cédula/DUI, o un ID autoincrementable (`id_usuario`). Dos usuarios jamás tendrán la misma PK.
🔗 Clave Foránea (FK - Foreign Key): Es un campo en una tabla que hace referencia o apunta a la Clave Primaria (PK) de otra tabla. Es el puente que une y relaciona ambas entidades.
